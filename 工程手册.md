## 2. 系统是什么

VS_AI 不是单点脚本，而是一条完整情报生产链：

```text
collect
-> dedup / classify / score
-> topic builder
-> AI analysis / summary / forecast
-> render
-> publish
-> archive
-> pages / review queue / ops artifacts
```

当前主赛道：

- AI / LLM / Agent
- AI x Electronics

当前稳定交付面：

- `data/reports/` 最新报告
- `docs/` 公开站
- 飞书消息


## 6. 发布级流程

标准本地链：

```powershell
$env:PYTHONPATH = "src"
pwsh ./scripts/check-workflows.ps1 -Profile full
python -m pytest tests -q
python -m auto_report.cli evaluate-prompts --dataset config/prompt_eval/baseline-v1.json
$env:AI_DISABLE_LLM = "true"
$env:AUTO_PUSH_ENABLED = "false"
python -m auto_report.cli run-once --publication-mode reviewed
python -m auto_report.cli build-pages
python -m auto_report.cli build-ops-dashboard
python -m auto_report.cli build-source-governance
python -m auto_report.cli build-review-queue
```

推送与远端确认：

```powershell
git push origin main
gh run list --workflow collect-report.yml --branch main --limit 5
gh run view <run-id> --json status,conclusion,url,updatedAt
```

执行纪律：

- 本地通过后再 push
- 远端 run 记录 run id 即可，不把 Actions 当日常排错回路
- 只有阶段完成、发布确认或用户明确要求时才额外手动触发 workflow

## 9. 建库经验与防踩坑

### 9.1 工程原则

- 先把 artifact 做出来，再做自动化；没有 artifact 的自动化很难交接
- 真相文件必须少而稳定，当前唯一运行态真相就是 `data/state/run-status.json`
- 阶段收口要包含代码、文档、验证，不要只改代码
- 新能力优先 sidecar 化、配置化、可回滚化

### 9.2 Windows 本机清理坑

- 跑 `python -m pytest tests -q` 前先确认 `$env:PYTHONPATH = "src"`，否则会出现成片 `ModuleNotFoundError`
- `.pytest_cache` 可直接删除重建；如果 `.pytest_*`、`.codex-pytest-temp*`、`pytest-tempdir*` 在父目录里 owner 显示成 `...` 且 `Access is denied`，说明目录 ACL 已污染
- 非提升态 PowerShell 通常只能清掉 `.pytest_cache`；其余目录请用管理员 PowerShell 执行 `pwsh ./scripts/cleanup-ignored-temp-acl.ps1`

### 9.2 来源治理经验

- 不要为了追求 RSS 形态，破坏一个已经验证可用的官方 listing
- `candidate-updates.json` 是接近执行的清单，不是无脑自动应用
- blocked 源先做证据链，再决定是替换、watch 还是禁用
- `requests` 不通但 `curl/browser` 可通，通常说明是接入策略问题，不是“这个源彻底不能用”

### 9.3 发布与运维经验

- `workflow_dispatch` 之前先 push，否则跑不到本地修复
- GitHub Actions 是最终确认，不是主调试回路
- 远端 run 触发后先记录 run id，再继续本地推进
- reviewed 轨本地验收时默认：

```powershell
$env:AI_DISABLE_LLM = "true"
$env:AUTO_PUSH_ENABLED = "false"
```

### 9.4 本地清理经验

优先可删目录：

- `.codex-pytest-temp*`
- `.codex-tmp`
- `.pytest_cache`
- `.pytest-codex-*`
- `.pytest-tmp`
- `codex_pytest_temp`
- `pytest-tempdir*`
- `out/`
- `output/`
- `.superpowers/`

不要误删：

- `data/`
- `docs/`
- `src/`
- `tests/`
- `.github/`
