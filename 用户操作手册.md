## 2. 最常用命令

### 生成当天日报

```powershell
python -m auto_report.cli run-once
```

### 生成人工复核版

```powershell
python -m auto_report.cli run-once --publication-mode reviewed --reviewer <name> --review-note <text>
```

如果暂时没有复核信息：

```powershell
python -m auto_report.cli run-once --publication-mode reviewed
```

### 重建页面

```powershell
python -m auto_report.cli build-pages
```

### 补报

```powershell

```

### 推送诊断

```powershell

python -m auto_report.cli diagnose-delivery --mode canary --send
python -m auto_report.cli diagnose-delivery --mode full-report --send
```

## 3. 一次运行后看什么

最重要的结果文件：

- `data/state/run-status.json`
- `data/reports/latest-summary.md`
- `data/reports/latest-summary-reviewed.md`
- `docs/index.html`

如果你只看一个文件，优先看：

- `data/state/run-status.json`

## 4. `run-status.json` 怎么看

最值得看的字段：

- `publication_mode`
- `review`
- `delivery_results`
- `risk_level`
- `ai_metrics`
- `source_health`
- `source_stats.report_topics`

快速判断逻辑：

- 这次有没有真正发出去：看 `delivery_results.successful_channels`
- 哪个渠道失败了：看 `delivery_results.failed_channels`
- 是不是 reviewed：看 `publication_mode` 和 `review`
- 来源最近稳不稳：看 `source_health`

## 5. PushPlus / ClawBot 使用提醒

如果你使用的是 `clawbot`：

- 后台必须配置：
  - `PUSHPLUS_TOKEN`
  - `PUSHPLUS_SECRETKEY`
  - 安全 IP 白名单
- 微信侧必须保持 ClawBot 已激活

当前仓库已经支持判断：

- `haveContextToken=1`
  - ClawBot 当前有有效对话上下文
- `delivery_status=2`
  - 本条消息已成功送达

如果微信收不到，先检查：

1. 最近有没有主动给 ClawBot 发过消息
2. PushPlus 开放接口是否开启
3. `secretKey` 是否正确
4. 安全 IP 白名单是否还是当前公网 IP

## 6. 高频场景

### 场景 A：我只想看今天会产出什么

```powershell
python -m auto_report.cli run-once
```

看：

- `data/reports/latest-summary.md`
- `data/state/run-status.json`

### 场景 B：我要生成 reviewed 轨

```powershell
python -m auto_report.cli run-once --publication-mode reviewed
```

看：

- `data/reports/latest-summary-reviewed.md`
- `data/state/run-status.json`

### 场景 C：我只更新了模板或页面，不想重新采集

```powershell
python -m auto_report.cli render-report --publication-mode auto
python -m auto_report.cli build-pages
```

### 场景 D：我怀疑某天漏报了

```powershell
python -m auto_report.cli backfill --target-date YYYY-MM-DD
```

### 场景 E：workflow 是绿的，但我不放心

不要只看 Actions 颜色。

先看：

- `data/state/run-status.json > delivery_results`

### 场景 F：本地修好了，但线上没带上修复

原因通常是：

- `workflow_dispatch` 只跑远端已 push 版本

正确顺序：

1. 本地验证
2. push
3. 再触发线上 workflow
