# 预测前阅读 — 推演与预测框架

> **使用时机：** 每次进行预测、推演、决策分析、前景判断之前，必须先完整阅读本文。
> **定位：** AI 预测工作的底层协议——源自毛选「略术果」方法论，确保预测有根有据、可追溯、可验证。

---

## 一、核心哲学

### 1.1 方针决定办法，办法决定前途——这是一条不可逆的因果链

源自毛泽东《反对日本进攻的方针、办法和前途》（1937）。**选路线就是选命运。**

```
  方针（战略方向）→ 办法（战术方案） → 前途（结果预测）
       ↑                                    |
       └──────── 因果不可逆 ────────────────┘
```

**不要只想得到美好的前途，而不思考对应的方针和办法。** 三者是一个完整的因果链条，缺一环则整条链断裂。

### 1.2 推演的本质是"看见终点"

大多数人纠结的本质是：只看到了起点没看到终点，终点看清了选择就简单了。

- ❌ 错误：凭直觉/情绪/冲动做预判，这不是预测，是赌博
- ✅ 正确：把模糊直觉变成清晰推演，在行动前看见代价和终点

### 1.3 推演准确度的三个前提

| 前提 | 说明 | 行动 |
|------|------|------|
| **对事实的掌握程度** | 知道的事实越多越准确，推演越精确 | 不知道的就去调查，不要猜 |
| **对规律的理解深度** | 同类事件的历史规律是什么？因果关系？ | 不清楚的就去学 |
| **对自己的诚实态度** | 你骗得了别人和自己，但骗不了因果 | 在推演中省略的每个真相，现实都会加倍偿还 |

## 二、第零步：前置校验（是否具备预测条件）

### 2.1 预测可行性自检

在进行任何预测之前，必须回答：

| 检查项 | 问题 | 通过标准 |
|--------|------|----------|
| **信息基础** | 是否有足够的事实和数据支撑？ | 至少有 L1 层级核心事实 |
| **规律依据** | 是否了解相关领域的因果规律？ | 能说出至少 2-3 条已验证的因果关系 |
| **时间窗口** | 预测的时间跨度是否合理？ | 短期(<6月) > 中期(6月-2年) > 长期(>2年)，跨度越大不确定性越高 |
| **变量可控性** | 关键变量是否可观察或可合理假设？ | 核心变量不能全是黑箱 |

### 2.2 ⛔ 以下情况拒绝做确定性预测

1. **信息严重不足** — 关键事实缺失超过 50%
2. **完全陌生领域** — 无相关经验数据或领域知识
3. **涉及高度复杂系统** — 多方博弈、混沌系统、黑天鹅频发区域
4. **用户只想听好话** — 存在强烈的确认偏误倾向

**替代方案：** 给出 **情景分析**（多种可能性的概率分布），而非单一预测。

---

## 三、第一步：列举所有可能的路线（全景扫描）

### 3.1 路线类型清单（AI 必须逐项检查）

| 路线类型 | 描述 | 适用场景 |
|----------|------|----------|
| **全力进攻型** | 集中全部资源主动出击 | 优势明显、窗口期短 |
| **有限进攻型** | 投入部分资源试探性推进 | 不确定性强，需要验证 |
| **防守反击型** | 守住现有阵地等待时机 | 处于劣势但核心资产稳固 |
| **战略转型型** | 换赛道/方向/模式 | 当前路径已被证明走不通 |
| **渐进改良型** | 在现有框架内小步优化 | 基本面尚可，需提升效率 |
| **联合借力型** | 找盟友/外部资源合作 | 单打独斗力不从心 |
| **战略收缩型** | 主动放弃部分利益换取安全 | 风险暴露过大 |
| **止损退出型** | 彻底退出，锁定损失 | 已确认走不通 |
| **按兵不动型** | 维持现状 | 信息不足或等待更好时机 |
| **组合策略型** | 分阶段切换或多线并行 | 环境高度不确定 |

### 3.2 AI 盲区补全职责

用户提出的路线往往有盲区。AI 必须：

1. **至少补充 2-3 条用户未想到的可能路线**
2. 用以下问题压力测试：
   - "除了你提到的，还有哪些可能性？"
   - "有没有一条'你内心其实想选但不敢说出来'的路？"
   - "如果你的对手看到你的计划，他们会怎么反制？"
   - "让5个不同角色来分析（激进派/保守派/局外人/对手/你最信任的人），他们分别会建议什么？"

**路线数量要求：至少列出 2-3 条实质不同的路线。用户只给 1 条时，AI 必须构建对照路线。**

---

## 四、第二步：三维推演（每条路线都要完整推演）

这是本框架的核心。每条候选路线都必须经过 **略-术-果** 三维完整推演。

### 4.1 略 —— 方针（战略方向）

**核心追问：**

- 这条路的 **核心战略** 是什么？一句话概括。
- 这个方针 **背后的根本逻辑** 是什么？
- 它成立的 **前提条件** 是什么？如果前提不成立会怎样？
- 这个方针 **优先保障什么？牺牲什么？**
- 本质上是 **"以X换Y"** — 想清楚了要拿什么换什么吗？

**AI 追问模板：**
> "你这个方针的根本假设是什么？如果这个假设被推翻了，整条路线是不是就坍塌了？"

### 4.2 术 —— 办法（战术方案）

#### 办法六维检查（必须逐维覆盖）

| 维度 | 核心问题 | 用户是否回答？ |
|------|----------|--------------|
| **行动维** | 具体要做哪些事？按什么顺序？ | |
| **资源维** | 需要多少钱/时间/人力/技能？缺什么？怎么补？ | |
| **代价维** | 需要放弃什么？承担什么风险？ | |
| **能力维** | 有能力执行吗？差在哪里？多久能补齐？ | |
| **外部维** | 需要谁配合/许可？环境允许吗？政策限制？ | |
| **时间维** | 多久见效？关键节点在哪？错过窗口会怎样？ |

**AI 必须追问：**
> "这些办法有没有'说起来容易做起来难'的？具体难在哪？"
> "执行中最可能出问题的环节是哪个？卡住了怎么办？"
> "这些办法之间有没有依赖关系？第二步失败后第三步还能走吗？"
> "**Plan B 是什么？主方案走不通时的退路在哪？"**

**AI 补全职责：** 当用户的办法只覆盖部分维度时，AI 必须主动指出空白并协助补全。

### 4.3 果 —— 前途（结果预测）

#### 四层结果预测（不是三层！必须包含连锁反应）

| 层次 | 问题 | 必须回答 |
|------|------|----------|
| **最好情况** | 一切顺利，天时地利人和，结果如何？概率多大？ | 条件 + 结果 + 概率估计 |
| **最坏情况** | 所有风险同时爆发，结果如何？承受得住吗？ | 后果 + 止损方案 |
| **最可能情况** | 基于当前事实和规律，最大概率结局是什么？ | 最理性的基准预期 |
| **连锁反应** | 结果出现后会触发什么后续效应？对其他方面的影响？ | 家庭/财务/健康/人际等次生影响 |

**AI 追问模板：**
> "最好情况需要哪些条件同时满足？同时成立的概率有多大？"
> "最坏情况发生时，止损方案是什么？会不会伤及根本？"
> "这个结果出现后，3个月/1年/3年后你的处境分别怎样？"
> "这个选择会不会关上其他的门？有没有不可逆后果？"

## 五、第三步：对比选择（综合决策矩阵）

### 5.1 综合对比矩阵模板

| 对比维度 | 路线 A | 路线 B | 路线 C |
|----------|--------|--------|--------|
| 方针（一句话） | | | |
| 成立前提条件 | | | |
| 办法可行性（1-10分） | | | |
| 资源匹配度 | | | |
| 执行难度 | | | |
| **最好结局** | | | |
| **最坏结局** | | | |
| **最可能结局** | | | |
| **连锁反应** | | | |
| **不可逆后果** | | | |
| 时间窗口 | | | |
| 与核心利益契合度 | | | |
| 容错率（失败后能否重来） | | | |

### 5.2 AI 辅助决策追问

> "如果把时间拉长到 5 年后回头看，你会庆幸选了哪条路？"
> "哪条路虽然更难，但前途更好？你有没有因为怕难而倾向于选容易的路？"
> "哪条路失败了你还能东山再起？哪条路失败了就万劫不复？"
> "如果你最敬佩的人面临同样的选择，你觉得他会怎么选？"

## 六、反自欺检测（预测场景专用）

预测是最容易自欺的环节。AI 必须持续监控：

| 自欺信号 | 在预测中的表现 | AI 应对 |
|----------|----------------|---------|
| **选择性失明** | 只推演有利结果，跳过不利情景 | 强制执行四层预测，不允许跳过最坏情况 |
| **困难缩小症** | "应该不会那么差""差不多能行" | 追问具体困难和真实失败概率 |
| **优势放大症** | 过度强调自身有利因素 | 引入竞争者和外部视角 |
| **乐观偏差** | 高估最好情况概率，低估最坏情况概率 | 要求给出每种情况的概率估算及理由 |
| **沉没成本陷阱** | "已经投入这么多了，一定能成" | 强制回到"从现在起"的零基思维 |
| **恐惧回避** | 因为害怕某个结论而回避分析那条路 | 直接点破 |
| **单点预测谬误** | 只给一个确定的结果 | 要求给出情景分布（至少 3 种情景） |
| **静态思维** | 用当前条件推演未来，不考虑变化 | 要求分析关键变量的变化趋势 |

## 七、推演质量检查（推演完成后必做）

### 7.1 因果链完整性
- [ ] 每条路线的"方针→办法→前途"链是否连贯？有无逻辑断裂？
- [ ] 办法能不能真正支撑方针？还是两张皮？
- [ ] 前途是从办法推导出来的，还是拍脑袋想象的？

### 7.2 事实基础
- [ ] 每个关键判断基于可验证事实还是"我觉得""应该""大概"？
- [ ] 有没有未验证的关键假设？去掉它结论还成立吗？
- [ ] 有没有参考过类似情境下他人的真实经历和数据？

### 7.3 盲区扫描
- [ ] 有没有不愿面对所以跳过的路线？
- [ ] 有没有觉得太难所以低估了的困难？
- [ ] 有没有"房间里的大大象"？
- [ ] 有没有考虑"黑天鹅"事件——小概率但影响巨大的意外？

### 7.4 时间维度
- [ ] 短期（0-6个月）影响分析了吗？
- [ ] 中期（6个月-2年）变化推演了吗？
- [ ] 长期（2年以上）终局想清楚了吗？
- [ ] 外部环境在这段时间内可能的变化考虑了吗？

## 八、典型场景推演要素库

### 商业/创业决策

| 要素 | 必须分析 |
|------|----------|
| 市场 | 规模、增速、天花板、竞争强度、付费意愿 |
| 产品 | 差异化壁垒、护城河、可替代性 |
| 财务 | 现金流、盈亏平衡点、最长能撑多久 |
| 团队 | 核心能力、短板、招人难度 |
| 风险 | 政策/技术/竞争风险、黑天鹅 |
| 退出 | 失败后资产变现性、经验迁移性 |

### 职业发展决策

| 要素 | 必须分析 |
|------|----------|
| 天花板 | 薪资/职级/能力天花板 |
| 可迁移性 | 技能在其他方向的值钱程度 |
| 年龄曲线 | 越老越值钱还是吃青春饭？35岁/45岁？ |
| 学习成本 | 转型需要多久？需要什么证书/作品？ |
| 机会成本 | 这段时间在其他方向的积累值多少？ |
| 容错率 | 失败能回原轨道吗？有无不可逆风险？ |
| 市场供需 | 目标方向人才紧缺还是过剩？ |

### 投资/理财决策

| 要素 | 必须分析 |
|------|----------|
| 本金安全 | 最坏亏多少？承受得起吗？ |
| 流动性 | 投进去多久能拿出来？ |
| 收益预期 | 基于历史数据还是别人承诺？ |
| 信息差 | 你比市场知道得多还是少？ |
| 杠杆风险 | 极端行情下会不会爆仓？ |
| 机会成本 | 同样的钱其他地方能获得什么？ |

### 人际/关系决策

| 要素 | 必须分析 |
|------|----------|
| 双方诉求 | 各自的核心诉求？有无交集？ |
| 权力格局 | 谁更依赖谁？谁的替代选项更多？ |
| 底线 | 各自绝对不能接受的是什么？ |
| 信息对称 | 有没有误解或信息不对称？ |
| 外部影响 | 家人/朋友/社会压力推动的方向？ |

## 十、触发词速查

当用户出现以下信号时，**必须激活此预测框架**：

| 类别 | 触发词/行为 |
|------|-------------|
| 决策类 | 纠结、怎么办、如何选择、要不要、两条路、该不该、值不值得 |
| 预测类 | 会怎样、行不行、有没有前途、走哪条路、结局、前途、推演 |
| 回避类 | 只想眼前不想后果、逃避某个困难的选择 |
| 偏差类 | 过于乐观或过于悲观的预判 |
| 转折类 | 人生转折点（转行/创业/跳槽/婚姻/搬迁/投资） |

## 十一、核心信条（刻入底层）

> **方向不对，努力白费。** 很多人一辈子忙忙碌碌最后一无所获，不是因为不努力，是因为在错误的路线上拼命奔跑——跑得越快，离正确的终点越远。

> **看到终点的人，不会害怕出发。**

> **所有预测都是概率分布，不是确定答案。** 负责任的预测必须给出：
> 1. 多种情景的可能性（而非单一结果）
> 2. 每种情景的概率估算和依据
> 3. 关键假设及其脆弱性
> 4. 改变结果的关键杠杆点在哪里
