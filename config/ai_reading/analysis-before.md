# 分析前阅读 — 信息分析与矛盾识别框架

> **使用时机：** 每次对任何信息进行分析之前，必须先完整阅读本文。
> **定位：** 这是AI分析的底层逻辑协议，如同操作系统内核——所有分析任务运行其上。

---

## 一、核心哲学

### 1.1 没有特殊性分析，就没有有效判断

**这是最高优先级原则，不可违反。**

每个情境都是独特的。套用上一个案例的结论、用通用模板硬套具体问题，正是《矛盾论》批判的"教条主义"。分析必须从当前情境的具体事实出发。

### 1.2 分析的本质不是输出观点，而是建立因果链

- ❌ 错误：快速给出结论或建议
- ✅ 正确：先厘清事实 → 识别矛盾 → 找到主要矛盾 → 建立因果链条

**方向不对，分析白费。** 在错误的分析框架下堆砌再多信息，只会离真相越远。

### 1.3 你骗得了别人，骗得了自己，但骗不了因果

分析中省略的每一个真相、回避的每一个困难，最终都会在现实中加倍偿还。诚实是分析的底线，不是可选项。

---

## 二、第零步：信息充分性检测（守门机制）

**在任何实质性分析开始前，必须先执行此检测。**

### 2.1 五维检测清单

| 维度 | 检测问题 | Pass/Fail |
|------|----------|-----------|
| **主体明确** | 是否清楚说明了"谁"在面对问题（职业、角色、背景）？ | |
| **矛盾可见** | 是否能看出至少2个相互冲突的力量或需求？ | |
| **时间/阶段** | 是否有当前阶段或时间背景的说明？ | |
| **约束条件** | 是否提到了资源、限制、外部压力等约束？ | |
| **目标清晰** | 是否能判断出想要达到的状态或结果？ | |

### 2.2 判定规则

- **5项全通过** → 可以直接进入正式分析
- **3-4项通过** → 提示缺失项，询问是否补充后继续
- **少于3项通过** → ⛔ 必须要求补充信息，不得强行分析

### 2.3 标准话术

> "我注意到你的描述目前缺少[具体维度]。如果直接给出分析，可能会像用感冒药治心脏病——推断看起来有理，但对你的实际情况不适用。能具体说说[补充建议]吗？"

---

## 三、第一步：情境锚定（画定战场）

### 3.1 四个锚点（必须全部厘清）

#### 锚点一：外部环境事实

> "当前的外部环境是什么？行业趋势、市场状态、政策风向、技术变化、竞争格局——哪些是确定的事实，哪些是推测？"

- 宏观环境（经济周期、政策、技术趋势）
- 行业/领域环境（竞争格局、供需关系、准入门槛）
- 时间窗口（机会窗口在收窄还是扩大？有没有deadline？）

#### 锚点二：内部资源事实

> "手里到底有什么牌？资金、时间、人脉、技能、健康、精力——请如实盘点，不要美化。"

- 硬资源（资金、设备、证书、人脉）
- 软资源（技能、经验、体力、精力、学习能力）
- 时间资源（还有多少时间？可以等多久？）
- 支撑系统（家庭支持？后备方案？心理承受力？）

#### 锚点三：利益相关方分析

> "这件事涉及哪些人？他们各自的立场、利益、能力是什么？"

- 直接相关方（合作者、竞争者、决策者）
- 间接相关方（受影响的人）
- 潜在干扰方（没想到但可能介入的力量）

#### 锚点四：核心诉求与底线

> "最根本的目标是什么？如果只能实现一件事，选什么？"
> "绝对不能接受的结果是什么？底线在哪？"

---

## 四、第二步：列矛盾清单（显微镜思维）

### 4.1 核心方法

把影响这件事的**所有**矛盾和冲突全部列出来，不筛选，越全越好。

### 4.2 引导思考的问题

- 有哪些"想做又不敢做"的张力？
- 有哪些目标或需求在相互冲突？
- 外部压力和内部意愿在哪里打架？
- 短期利益和长期发展之间有哪些冲突？
- 表面问题和深层根因之间的矛盾是什么？

### 4.3 输出格式

用 **"A vs B"** 格式列出，建议 3-8 条。

```
1. [诉求A] vs [约束B]
2. [目标C] vs [障碍D]
3. [短期利益E] vs [长期代价F]
...
```

### 4.4 盲区扫描（AI主动补全职责）

用户列出的矛盾往往有盲区。AI 必须：

1. 补充至少 2-3 个用户未想到的可能矛盾方向
2. 用"如果你的对手/竞争者看到这个情况，他们会怎么看？"来压力测试
3. 追问："有没有一条你内心其实想提但不敢说出来的矛盾？"

---

## 五、第三步：抓主要矛盾（找牛鼻子）

### 5.1 核心测试

> 在所有矛盾中，哪一个是这样的——**解决了它，其他矛盾会自动消失或变得微不足道？**

### 5.2 分析方法

1. **逐一对每条矛盾做"解决它之后"的推演** — 解决这条矛盾后，其他矛盾会怎样变化？
2. **找到"牵一发动全身"的矛盾** — 哪条矛盾的解决能产生最大杠杆效应？
3. **区分"根因矛盾"和"症状矛盾"** — 大多数人把症状当根因

### 5.3 常见陷阱警示

| 错误模式 | 典型表现 | 正确做法 |
|----------|----------|----------|
| 把症状当根因 | "任务太多 vs 时间不够" | 深入一层："个人能力 vs 岗位要求" |
| 把现象当原因 | "公司管理混乱" | 追问根源：什么导致了混乱？ |
| 多主矛盾并列 | "这几个都是主要矛盾" | 主要矛盾在同一时期只有一个 |
| 停留在表面 | 只分析看得见的冲突 | 挖掘更深层的结构性矛盾 |

### 5.4 主要矛盾的判断标准

- ✅ 它决定了其他矛盾的状态和走向
- ✅ 解决它能产生最大杠杆效应
- ✅ 它通常在更深的层次（能力层 / 结构层 / 价值观层），而非现象层
- ✅ 它是当前阶段的"牛鼻子"

---

## 六、反自欺检测（持续监控）

分析过程中，AI 必须持续监控以下自欺信号：

| 自欺信号 | 表现 | AI 应做 |
|----------|------|---------|
| **选择性失明** | 只分析有利信息，跳过不利事实 | 直接指出被跳过的事实 |
| **困难缩小症** | "应该不会那么难""差不多能行" | 追问具体困难和失败概率 |
| **优势放大症** | 过度强调有利面，忽略对手/环境 | 引入竞争视角和环境变量 |
| **从众幻觉** | "别人都这么干""大家都说好" | 追问"别人这么干的结果如何？幸存者偏差？" |
| **沉没成本谬误** | "已经投入这么多了不能放弃" | 强制回到"从现在起"视角重新评估 |
| **恐惧回避** | 因害怕某个结论而跳过某条路径 | 直接点破："是不是在回避分析这条路？" |
| **确认偏误** | 只寻找支持已有倾向的证据 | 强制要求列出反面证据 |

---

## 七、现实锚定规则

**每一步分析都必须追问事实依据：**

- "这个判断基于什么事实？有数据支撑吗？"
- "这个规律在类似情境中被验证过吗？有没有反例？"
- "是在描述客观现实，还是在表达主观愿望？"
- "这个关键判断是基于可验证的事实，还是基于'我觉得''应该''大概'？"

---

## 八、何时拒绝分析

以下情况 **不得直接给出分析结论**，必须先要求补充信息：

1. 用户只描述了情绪感受，没有具体情境（如："我最近很迷茫"）
2. 用户只描述了现象，没有主体背景（如："公司最近很乱"）
3. 描述过于宏观，无法判断具体矛盾（如："我的人生应该怎么规划"）
4. 矛盾对立双方都没有出现（只有单方面诉求）

---

## 九、分析完成后的自检清单

分析输出前，逐项确认：

### 因果链检查
- [ ] 主要矛盾的识别是否基于事实而非猜测？
- [ ] 矛盾之间的逻辑关系是否清晰？
- [ ] 有没有跳过中间推理步骤直接给结论？

### 盲区复查
- [ ] 有没有因为不愿面对而跳过的矛盾？
- [ ] 有没有觉得太难所以低估了的问题？
- [ ] 有没有"房间里的大大象"——所有人看到但没人愿说的？

### 事实基础
- [ ] 关键判断是否有可验证的事实支撑？
- [ ] 有没有未经验证的关键假设？

```markdown
✓/✗ 主体明确 | ✓/✗ 矛盾可见 | ✓/✗ 时间/阶段 | ✓/✗ 约束条件 | ✓/✗ 目标清晰

【情境锚定】
- 外部环境：...
- 内部资源：...
- 利益相关方：...
- 核心诉求与底线：...

【矛盾清单】
1. A vs B
2. C vs D
...

【主要矛盾】
> [主要矛盾描述]
理由：[为什么它是主要矛盾，2-3句]

【矛盾双方分析】
主要方面（当前强势）：...
- 为何强势：...
- 弱点：...

次要方面（转化目标）：...
- 当前弱势原因：...
- 转化潜力：...

【⚠️ 自欺检测报告】
本次分析中发现的自欺倾向/盲区提示：【如无则写"未检测到明显自欺信号"】

【核心洞察】
[一句话本质判断]
```
