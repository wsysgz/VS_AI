# 领域简报生成规范（V1）

在输出单领域简报时，保持以下结构：

1. `title`
   - 使用领域展示名，保持稳定，不自造别名。
2. `one_line_core`
   - 用一句话说明该领域今天最值得关注的主判断。
3. `executive_summary`
   - 输出 2-4 条要点，优先保留工程进展、风险变化和节奏拐点。
4. `signals`
   - 只保留该领域内最重要的主题，确保标题、链接、why_it_matters 可直接阅读。

写作要求：

- 不把采集器原始报错直接写进公开简报。
- 不重复总报里的冗余措辞，强调本领域特有变化。
- 遇到证据不足时，明确标注仍需继续观察。
